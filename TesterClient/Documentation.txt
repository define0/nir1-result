TesterClient.exe
Возможно лучшая программа тестирования MCA сервера

usage:
TesterClient.exe <test> [params]

<test> API запрос, который тестируем
	"load" - /api/v1/load

[params]: (может быть любого регистра)
/RUNID:<n> 			- <n>=идентификатор программы, например для лога (по умолчанию: 1)
/THREADS:<n> 		- <n>=количество потоков тестирования (по умолчанию: 1)
/ENTRIES:<n> 		- <n>=количество генерируемых записей за тест (по умолчанию: 10)
/HOST:<host>		- <host>=сервер, к которому подключается программа (по умолчанию: iamnotaprogrammer.ru)
/PORT:<port> 		- <port>=порт, к которому подключается программа (по умолчанию: 8880)
/CHUNKED 			- передавать по частям или нет (Transfer-Encoding: chunked) 
/VERBOSE 			- вести ли подробное логирование 
/DEBUG 				- вести ли отладочную информацию и отображать все запросы
/KEEPALIVE:<n> 		- <n>=значение Keep-Alive (по умолчанию: 60)
/MINSENDINGDATA:<n> - <n>=минимальное количество байт, передаваемой за чанк (по умолчанию: 100)
/MAXSENDINGDATA:<n> - <n>=максимальное количество байт, передаваемое за чанк (по умолчанию: 200)
/MINSLEEPTIME:<n> 	- <n>=минимальное время задержки, между передачей чанков (по умолчанию: 1)
/MAXSLEEPTIME:<n>	- <n>=максимальное время задержки, между передачей чанков (по умолчанию: 2)

example:
TestClient.exe load /RunID:228 /Threads:10 /Entries:1337 /Host:iamnotaprogrammer.ru /Port:8880 /Chunked
